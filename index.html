¿Qué es Linux?

Linux es una familia de sistemas operativos de código abierto basados en el kernel Linux, lanzado originalmente el 17 de septiembre de 1991 por Linus Torvalds. Técnicamente, Linux es únicamente el kernel (núcleo) del sistema operativo; el sistema completo —con todas sus herramientas, compiladores y utilidades— se denomina correctamente GNU/Linux, ya que combina el kernel con el software del proyecto GNU de Richard Stallman. Es un sistema operativo diseñado para aprovechar al máximo las capacidades del hardware, con características fundamentales como el multiprocesamiento, la multitarea y el soporte multiusuario. Hoy en día, Linux impulsa desde teléfonos inteligentes hasta supercomputadoras, servidores web, satélites y prácticamente toda la infraestructura de internet moderna.

Historia y Origen


Raíces en UNIX y el proyecto GNU (1969–1991)

Todo comenzó en 1969, cuando un grupo de ingenieros de AT&T Bell Labs desarrolló UNIX, un sistema operativo portátil, multitarea y multiusuario. Durante la siguiente década, UNIX se convirtió en el estándar de facto en entornos académicos y corporativos. En 1983, el programador Richard Stallman inició el Proyecto GNU (GNU’s Not Unix) con un objetivo ambicioso: crear un sistema operativo completamente libre, compatible con UNIX. Para 1985 fundó la Free Software Foundation (FSF) y en 1989 publicó la Licencia Pública General GNU (GPL), que garantiza las cuatro libertades fundamentales del software libre: usar, estudiar, modificar y redistribuir. El proyecto GNU desarrolló compiladores (GCC), editores de texto (Emacs), herramientas del sistema y muchos otros componentes esenciales. Sin embargo, faltaba el componente más crítico: el kernel.

Linus Torvalds y el nacimiento del kernel (1991)

En 1991, Linus Torvalds, un estudiante finlandés de ciencias de la computación de 21 años en la Universidad de Helsinki, comenzó a trabajar en un terminal virtual para su computadora personal con un procesador Intel 386. Su objetivo inicial era crear algo que le permitiera ejecutar simultáneamente varias sesiones de terminal. El 25 de agosto de 1991, Torvalds publicó en el grupo de noticias de Usenet comp.os.minix el famoso mensaje: “Estoy haciendo un sistema operativo libre (solo un hobby, no será grande ni profesional como GNU) para clones AT 386(486).” El 17 de septiembre de 1991, publicó la versión 0.01 del kernel en internet para que otros pudieran verlo y contribuir. En diciembre de ese año lanzó la versión 0.11, la primera capaz de ejecutarse como sistema operativo independiente.

Arquitectura del Sistema

La arquitectura de un sistema GNU/Linux es una estructura en capas bien definida, donde cada nivel de abstracción cumple una función específica.

El Kernel: Corazón del Sistema

El kernel es el componente central del sistema operativo. Es la capa de software responsable de asegurar que todos los programas y procesos tengan acceso a los recursos que necesitan del hardware (memoria RAM, CPU, disco, red) al mismo tiempo, gestionando que haya un acceso compartido y ordenado.

Tipo de Arquitectura del Kernel

El kernel de Linux implementa una arquitectura monolítica modular, que es una de las razones clave de su eficiencia:

Monolítico: todas las funciones principales (gestión de procesos, memoria, E/S, sistemas de archivos) corren en el espacio del kernel con acceso directo al hardware, lo que garantiza alto rendimiento.

Modular: incorpora características de los microkernels mediante módulos.

Cargables: (archivos .ko), que pueden insertarse o eliminarse dinámicamente en tiempo de ejecución sin necesidad de reiniciar el sistema ni recompilar el núcleo completo.

Componentes internos del kernel

Gestión de procesos: determina qué procesos pueden usar la CPU, cuándo y por cuánto tiempo. El planificador por defecto es el CFS (Completely Fair Scheduler), que asegura una distribución equitativa del tiempo de CPU. También soporta clases de tiempo real (SCHED_FIFO, SCHED_RR) para aplicaciones críticas.

Gestión de memoria: incluye paginación, asignadores de memoria (SLAB/SLUB), mapeos de usuario mediante mmap, memoria anónima y page cache. La separación entre kernel space y user space es fundamental para la seguridad y estabilidad del sistema.

Sistema de archivos virtual (VFS): proporciona una capa de abstracción que permite al kernel soportar múltiples sistemas de archivos (ext4, btrfs, xfs, ntfs, fat32) de forma transparente para las aplicaciones.

Controladores de dispositivos (drivers): actúan como traductores entre el hardware y los procesos. Pueden compilarse directamente en el kernel o como módulos cargables.

Stack de red: implementación completa del protocolo TCP/IP, soporte para IPv4 e IPv6, firewalling (netfilter/iptables/nftables) y comunicación entre procesos (IPC).

Llamadas al sistema (syscalls): la interfaz que permite a los programas del espacio de usuario solicitar servicios al kernel. Están definidas en el estándar POSIX.

El Shell: Interfaz de comandos

El shell es un programa que actúa como capa intermedia entre el usuario y el kernel. Interpreta los comandos del usuario y los traduce en llamadas al sistema. Existen varios tipos:

  • Bash (Bourne Again Shell): el más utilizado por defecto en la mayoría de distribuciones. Implementa un lenguaje de programación simple para shell scripts.
  • Zsh: con autocompletado avanzado, personalización mediante plugins (Oh My Zsh). Usado por defecto en macOS desde 2019.
  • Fish : shell con sintaxis amigable y resaltado de sintaxis en tiempo real.
  • Dash:shell ligero, compatible con POSIX, usado para scripts de sistema.
  • Ksh (Kron Shell): popular en entornos Unix corporativos.

El Bootloader

El proceso de arranque sigue estas etapas:

  1. BIOS/UEFI: verifica el hardware (POST) y busca el dispositivo de arranque.
  2. Bootloader: (GRUB2, systemd-boot, LILO): carga el kernel en memoria y pasa parámetros de arranque.
  3. Descompresión del kernel: se inicializa la MMU, la memoria y los controladores base.
  4. Montake del sistema de archivos raíz desde initramfs.
  5. Transición al espacio de usuario: se ejecuta init o systemd como proceso con PID 1.

Características Principales

Código abierto y licencia GPL

Linux está distribuido bajo la Licencia Pública General GNU (GPL v2), lo que significa que su código fuente está disponible para que cualquier persona lo vea, modifique y distribuya. Esta apertura ha permitido que miles de desarrolladores de todo el mundo contribuyan al proyecto, creando un ciclo virtuoso de mejora continua.

Multitarea y multiprocesamiento

Linux soporta verdadera multitarea preventiva: el kernel puede interrumpir cualquier proceso en cualquier momento para dar CPU a otro. Esto garantiza que ningún proceso monopolice el sistema. Además, soporta multiprocesamiento simétrico (SMP), aprovechando todos los núcleos del procesador de manera eficiente.

Multiusuario

Múltiples usuarios pueden acceder simultáneamente a los recursos del sistema de forma segura. Cada usuario tiene su propio directorio de inicio (/home/usuario) y solo puede acceder a los archivos y recursos para los que tenga permisos explícitos.

Seguridad robusta

Linux tiene una arquitectura de seguridad en múltiples capas:

  1. Control de acceso discrecional (DAC): sistema de permisos Unix (usuario, grupo, otros) con lectura, escritura y ejecución.
  2. SELinux (Security-Enhanced Linux): desarrollado originalmente por la NSA, implementa control de acceso obligatorio (MAC) con políticas granulares.
  3. AppArmor: Alternativa a SELinux, usada por Ubuntu y SUSE, define perfiles por aplicación.
  4. Capabilities: divide los privilegios del superusuario en unidades más pequeñas y manejables.
  5. Seccomp: permite filtrar las llamadas al sistema disponibles para cada proceso, reduciendo la superficie de ataque.
  6. Namespaces y cgroups: base de la tecnología de contenedores (Docker, Podman).

Portabilidad

El kernel de Linux se puede compilar para una enorme variedad de arquitecturas de hardware: x86, x86_64, ARM, ARM64, RISC-V, PowerPC, MIPS, SPARC y muchas más. Esto explica su presencia en dispositivos tan diversos como teléfonos Android, routers, televisores inteligentes, mainframes IBM y los supercomputadores más potentes del mundo.

Estabilidad y confiabilidad

Linux es famoso por su estabilidad. Muchos servidores Linux funcionan durante años sin necesidad de reiniciarse. Las actualizaciones de seguridad críticas pueden aplicarse en tiempo de ejecución mediante tecnologías como kpatch o livepatch, sin interrumpir el servicio.

Rendimiento

Gracias a su arquitectura monolítica, el kernel de Linux ofrece un rendimiento excepcional, especialmente en cargas de trabajo de servidor, bases de datos y computación de alto rendimiento (HPC). El planificador CFS y las optimizaciones continuas del kernel hacen que el sistema se adapte dinámicamente a las condiciones de carga.
