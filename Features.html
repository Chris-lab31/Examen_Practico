Cualidades/Caracteristicas de Linux


Features del Kernel

Multitarea preventiva

el kernel puede interrumpir cualquier proceso en cualquier momento para cederle CPU a otro. Ningún proceso puede monopolizar el sistema.

SMP (Symmetric Multiprocessing)

aprovecha todos los núcleos del procesador de forma eficiente. El planificador **CFS (Completely Fair Scheduler)** distribuye el tiempo de CPU de manera equitativa entre todos los procesos.

Módulos del kernel cargables

gracias a su arquitectura monolítica modular, los drivers y funcionalidades pueden cargarse y descargarse en caliente (sin reiniciar) mediante archivos `.ko`.

VFS (Virtual File System)

capa de abstracción que permite soportar múltiples sistemas de archivos (ext4, btrfs, XFS, NTFS, FAT32, ZFS) de forma transparente para las aplicaciones.

namespaces y cgroups

base de toda la tecnología de contenedores. Los namespaces aíslan procesos (red, PID, filesystem), y los cgroups limitan y monitorean el uso de recursos (CPU, RAM, I/O). Docker y Podman dependen completamente de esto.

eBPF (Extended Berkeley Packet Filter)

una de las features más poderosas y modernas del kernel. Permite cargar programas pequeños y verificados directamente en el kernel en tiempo de ejecución, sin modificarlo ni reiniciarlo. Se usa para observabilidad, seguridad, redes y profiling de rendimiento. Herramientas como `bpftrace`, Cilium y Falco lo usan extensamente.


Features de Seguridad

SELinux/ AppArmor

es un sistema de seguridad de aplicaciones que proporciona protección contra accesos no autorizados a los recursos de un sistema. Es un conjunto de reglas que se aplican a cada proceso y que determinan si un proceso puede acceder a un recurso determinado. Las reglas se pueden aplicar a nivel de sistema, de aplicación o de usuario. Las reglas se pueden aplicar a recursos de archivos, de red, de procesos y de sistema. Seccomp

filtra las llamadas al sistema disponibles para cada proceso. Docker lo usa para reducir la superficie de ataque de los contenedores.

Capabilities

divide los privilegios del superusuario en unidades más pequeñas y manejables, evitando que un proceso tenga más privilegios de los estrictamente necesarios.

Lockdown Mode

características como el modo lockdown representan victorias importantes en la seguridad de infraestructura crítica [TuxCare](https://tuxcare.com/blog/linux-evolution/) . Restringe incluso al usuario root para que no pueda comprometer la integridad del kernel en ejecución.

Kernel Address Space Layout Randomization (KASLR)

randomiza las direcciones de memoria del kernel en cada arranque, dificultando la explotación de vulnerabilidades.

Guard pages

una feature reciente agrega páginas de guardia (páginas sin acceso) alrededor de los stacks del kernel para prevenir ataques de desbordamiento de pila. [IBM](https://www.ibm.com/think/topics/linux)

ARM MTE (Memory Tagging Extension)

la arquitectura ARM recibe un impulso significativo con MTE, que detecta vulnerabilidades de corrupción de memoria como overflow y use-after-free. [IBM](https://www.ibm.com/think/topics/linux)


Rust en el Kernel

Desde la versión 6.1, Linux comenzó a integrar código en **Rust** como segundo lenguaje oficial del kernel, junto a C. Rust no elimina vulnerabilidades por completo, pero su diseño con seguridad de memoria reduce drásticamente la exposición a problemas como buffer overflows. [TuxCare](https://tuxcare.com/blog/linux-evolution/) En 2024 también se experimentó con un planificador de Linux escrito en Rust, mostrando resultados prometedores para el rendimiento en gaming. [ADSLZone](https://www.adslzone.net/reportajes/software/que-es-linux/)


Features de Rendimiento y Energia

PREEMPT_RT (Real-Time Linux)

el patchset PREEMPT_RT, que agrega capacidades de tiempo real con latencias sub-milisegundo, está siendo integrado al kernel mainline, lo que hace de Linux una opción top para robots industriales y vehículos autónomos. [Red Hat](https://www.redhat.com/en/topics/linux)

Gestión dinámica de energía

features como Intel Speed Select y AMD P-state permiten al kernel ajustar la frecuencia y voltaje de la CPU en tiempo real. Gobernadores con IA como `schedutil` usan machine learning para predecir cargas de trabajo y optimizar el consumo. [Red Hat](https://www.redhat.com/en/topics/linux)

io_uring

interfaz de I/O asíncrona de alto rendimiento introducida en Linux 5.1. Reduce drasticamente el overhead de las llamadas al sistema para operaciones de disco y red. Fundamental para servidores y bases de datos modernos.


Features de Sistema de Archivos

Bcachefs

un nuevo sistema de archivos COW (Copy-on-Write) que enfatiza la confiabilidad y robustez, con suma de verificación completa de datos y metadatos, replicación, compresión, cifrado e instantáneas. [ArrobaSolutions](https://www.arrobasolutions.com/30-anos-linux-detalles-exito/) Fue fusionado oficialmente en Linux 6.7.

Btrfs con RAID tree

Btrfs incorporó en Linux 6.7 el árbol de bandas RAID y mejoras de rendimiento en operaciones como el borrado. [ArrobaSolutions](https://www.arrobasolutions.com/30-anos-linux-detalles-exito/)


Features de Red y Virtualización

Netfilter / nftables

framework de firewall integrado en el kernel. `nftables` reemplaza a `iptables` con una sintaxis más limpia y mejor rendimiento.

KVM (Kernel-based Virtual Machine)

hipervisor integrado directamente en el kernel. Base de AWS, GCP y Azure para virtualizar millones de instancias. Linux 6.12 LTS incorporó optimizaciones de rendimiento en entornos virtualizados, haciéndolo ideal para nubes privadas y públicas. [MuyLinux](https://www.muylinux.com/2024/01/12/deseos-escritorio-linux-2024/)

SR-IOV

permite que una sola tarjeta de red física se presente como múltiples interfaces virtuales independientes al kernel, fundamental para datacenter y virtualización de red. [Red Hat](https://www.redhat.com/en/topics/linux)


Features para el Escritorio y Gaming

Wayland

protocolo gráfico moderno que reemplaza X11. Canonical tomó la decisión de hacer que Ubuntu use Wayland por defecto incluso en GPUs NVIDIA, habiendo anteriormente hecho fallback a Xorg/X11 por los problemas históricos de NVIDIA con Wayland. [Timetoast](https://www.timetoast.com/timelines/la-historia-de-linux-eb4c906e-c130-4665-8e59-efd23d4f7739)

Soporte RISC-V mejorado

con chips RISC-V más nuevos y rápidos emergiendo, y el soporte del kernel Linux para RISC-V mejorando rápidamente, esta arquitectura abierta gana terreno en IoT, embebido y servidores. [Timetoast](https://www.timetoast.com/timelines/la-historia-de-linux-eb4c906e-c130-4665-8e59-efd23d4f7739)
